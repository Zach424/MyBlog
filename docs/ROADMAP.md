# 开发路线图

状态定义：`done` 已实现且有验证证据；`partial` 已开始但未完成；`pending` 尚未开始。

| 阶段 | 状态 | 完成门槛 |
| --- | --- | --- |
| 1. 工程与归档基线 | done | 干净安装、构建通过、文档归档、独立提交 |
| 2. 内容与视觉系统 | partial | 内容 schema 与视觉方案已冻结，正式首页完成；Markdown 管线待实现 |
| 3. 博客核心功能 | partial | 首页可用；列表、文章、项目、专题、标签和关于页面待实现 |
| 4. 发布能力 | partial | 基础 SEO 和分享卡已完成；搜索、RSS、Sitemap 与草稿机制待实现 |
| 5. 上线候选 | pending | 桌面/移动关键流程、可访问性、性能和安全检查通过 |
| 6. 生产上线 | pending | Cloudflare 部署、域名、监控、回滚和维护手册完成 |

## 当前关键路径

1. 实现构建期 Markdown 读取、schema 校验、排序和派生索引；
2. 用至少 3 篇真实示例内容验证文章、项目、专题与标签 URL；
3. 让首页记录从内容管线读取，删除当前内联数据。

## 已知风险

- Vinext 仍需在项目范围内验证 Next.js 兼容边界。
- Markdown 构建方案尚未冻结，必须保证 Cloudflare 运行时不依赖文件系统。
- 动态主机元数据会让 Vinext 把 `/` 标记为 `Unknown` 路由；构建和 Worker 渲染均通过，部署验收时继续验证。
- 系统字体在不同操作系统上的字宽会略有差异，需要在浏览器验收中检查折行。
- 作者公开名称、简介、联系方式、域名和首批真实内容尚未确定。
