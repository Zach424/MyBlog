# 开发路线图

状态定义：`done` 已实现且有验证证据；`partial` 已开始但未完成；`pending` 尚未开始。

| 阶段 | 状态 | 完成门槛 |
| --- | --- | --- |
| 1. 工程与归档基线 | done | 干净安装、构建通过、文档归档、独立提交 |
| 2. 内容与视觉系统 | partial | 内容 schema 已冻结；三种视觉方案等待选择，核心页面设计基线尚未确定 |
| 3. 博客核心功能 | pending | 首页、列表、文章、项目、专题、标签、关于页面可用 |
| 4. 发布能力 | pending | 搜索、RSS、Sitemap、SEO、分享卡、草稿机制完成 |
| 5. 上线候选 | pending | 桌面/移动关键流程、可访问性、性能和安全检查通过 |
| 6. 生产上线 | pending | Cloudflare 部署、域名、监控、回滚和维护手册完成 |

## 当前关键路径

1. 从三个已生成的首页方向中选择或组合最终视觉语言；
2. 再实现共享布局与内容加载，避免先写页面后重构数据边界；
3. 用真实示例内容验证文章与项目页面，而不是使用占位 Lorem Ipsum。

## 已知风险

- Vinext 仍需在项目范围内验证 Next.js 兼容边界。
- Markdown 构建方案尚未冻结，必须保证 Cloudflare 运行时不依赖文件系统。
- 字体方案尚未验证许可、体积和大陆网络加载表现。
- 作者公开名称、简介、联系方式、域名和首批真实内容尚未确定。
