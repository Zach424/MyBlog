# 开发路线图

状态定义：`done` 已实现且有验证证据；`partial` 已开始但未完成；`pending` 尚未开始。

| 阶段 | 状态 | 完成门槛 |
| --- | --- | --- |
| 1. 工程与归档基线 | done | 干净安装、构建通过、文档归档、独立提交 |
| 2. 内容与视觉系统 | done | schema、构建期内容管线、Markdown 正文渲染、视觉方案与首页完成 |
| 3. 博客核心功能 | done | 首页、列表、文章、项目、专题、标签、关于与 404 路径通过集成测试 |
| 4. 发布能力 | done | SEO、分享卡、草稿过滤、搜索、RSS、Sitemap 与 robots 通过 Worker 集成测试 |
| 5. 上线候选 | pending | 桌面/移动关键流程、可访问性、性能和安全检查通过 |
| 6. 生产上线 | pending | Cloudflare 部署、域名、监控、回滚和维护手册完成 |

## 当前关键路径

1. 完成桌面/移动关键流程和键盘交互验收；
2. 检查可访问性、HTML 语义、性能预算与安全响应头；
3. 生成上线候选报告并冻结 Cloudflare 部署配置。

## 已知风险

- Vinext 仍需在项目范围内验证 Next.js 兼容边界。
- 当前未压缩 `dist/server` 总量约 2.66 MB；Cloudflare 部署前继续检查真实上传包与 Worker 限额。
- 动态主机元数据会让 Vinext 把 `/` 标记为 `Unknown` 路由；构建和 Worker 渲染均通过，部署验收时继续验证。
- 系统字体在不同操作系统上的字宽会略有差异，需要在浏览器验收中检查折行。
- GitHub 公开入口和首批内容已确定；最终域名、邮箱公开范围和其他联系方式仍待用户确认。
