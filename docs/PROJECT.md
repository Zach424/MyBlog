# 项目章程

## 目标

构建一个长期可维护的个人技术博客，将零散学习记录沉淀为可检索的知识，将项目经历沉淀为包含背景、选择、问题、结果和复盘的工程档案。

## 主要用户

1. 未来的自己：快速找回某个技术判断、问题和解决过程。
2. 同领域开发者：通过文章和项目复盘获得可复用的经验。
3. 潜在协作者：在较短时间内理解作者做过什么、如何思考。

## 产品承诺

首页的单一任务是：让访问者在一分钟内找到最近学习内容、代表项目和作者当前关注方向。

## 信息架构

| 区域 | 内容 | 第一版 |
| --- | --- | --- |
| 首页 | 定位、当前关注、精选项目、最近文章 | 必须 |
| 文章 | TIL、完整文章、系列文章 | 必须 |
| 项目 | 背景、架构、取舍、结果、复盘 | 必须 |
| 专题 | 按学习主题组织连续内容 | 必须 |
| 标签 | 横向技术索引 | 必须 |
| 关于 | 简介、技术栈、联系方式 | 必须 |
| 搜索 | 标题、摘要、正文和标签检索 | 上线前 |
| 评论 | GitHub Discussions / giscus | 上线后评估 |

## 内容类型

- `TIL`：短小、低发布摩擦的当天学习记录。
- `Article`：经过验证和组织的完整技术文章。
- `Series`：围绕同一学习目标的连续文章。
- `Project`：真实项目的工程记录和复盘。

## 第一版范围

### 包含

- 中文优先、响应式、浅色/深色模式；
- 本地版本管理的 Markdown 内容；
- 代码高亮、目录、阅读时间、更新时间；
- 标签、专题、搜索、RSS、Sitemap 和基础 SEO；
- Cloudflare 部署、自动构建和上线检查；
- 至少 3 篇示例文章和 1 个完整项目页面。

### 暂不包含

- 用户注册、自建评论数据库和管理后台；
- 推荐算法、付费内容、Newsletter 和多语言；
- 为低频需求引入独立后端服务。

## 成功标准

- 新建一篇内容后，五分钟内可以完成本地预览和发布提交。
- 每个公开页面具有稳定 URL、标题、摘要和分享信息。
- 手机和桌面端的关键导航、搜索及阅读流程通过浏览器验收。
- 构建、测试和部署可从干净安装重复执行。
- 所有开发轮次均具有对应归档和独立提交。
