# 设计基线

## 主题

暂定视觉概念为“工程工作日志”。它不是终端界面，也不是报纸模板，而是把真实的工程行为——提交、边注、状态和复盘——转化为信息结构。

视觉服务于两个目标：快速扫描学习进展，以及长时间舒适阅读技术内容。

## 设计对象

- 主题：Zach424 的学习记录与项目工程档案。
- 受众：未来的自己、开发者和潜在协作者。
- 首页任务：快速找到最近内容、代表项目和当前关注方向。

## 初始 Token 草案

以下 Token 是下一轮视觉方案的共同起点，不是最终锁定值。

| 角色 | 值 | 意图 |
| --- | --- | --- |
| Canvas | `#F2F6F7` | 冷调纸面，避开常见暖米色模板 |
| Ink | `#18263D` | 蓝黑工程墨水，承担正文和标题 |
| Muted | `#66727F` | 日期、标签和辅助信息 |
| Signal | `#E4572E` | 只用于状态、链接和关键动作 |
| Trace | `#B9D8DE` | 时间轨迹、边注线和选区 |
| Dark canvas | `#101820` | 深色阅读背景 |

## 字体角色

- 展示与少量章节标题：候选为“霞鹜文楷”，表达个人笔记感，但严格限制使用范围。
- 中文正文与界面：候选为 Noto Sans SC，优先清晰度和字符覆盖。
- 代码与状态信息：候选为 JetBrains Mono，用于真实的代码和提交信息。

字体许可、加载性能和中国大陆访问情况在定稿前验证；验证失败时使用本地字体栈，不阻塞内容发布。

## 布局概念

```text
┌─────────────────────────────────────────────────────┐
│ Zach424 / Engineering Notes        文章 项目 关于 搜索 │
├───────────────────────────┬─────────────────────────┤
│ 一句明确的个人定位          │ CURRENT FOCUS           │
│ 最近在解决什么问题          │ 真实的学习与项目状态      │
├───────── commit trace ─────┴─────────────────────────┤
│ 最近文章：日期、类型、标题、摘要、真实标签             │
├─────────────────────────────────────────────────────┤
│ 精选项目：目标 / 关键取舍 / 状态 / 复盘入口             │
└─────────────────────────────────────────────────────┘
```

文章页保持更安静：正文宽度约 720–800px，目录和边注只在空间允许时出现。

## 识别元素

“Commit Trace” 是唯一主要视觉风险：一条随内容延伸的工程轨迹，连接真实日期、文章类型和项目里程碑。它表达的是时间与积累，不作为无意义装饰。

页面其余部分保持克制，避免同时加入网格背景、大面积渐变、发光、玻璃效果和多套动画。

## 交互原则

- 键盘焦点始终清晰可见。
- 动画只用于页面进入或轨迹状态变化，并尊重 `prefers-reduced-motion`。
- 深色模式维持信息层级，而不是简单反色。
- 代码块、表格和长 URL 在手机端必须可用。

## 自我审查

最初曾考虑暖纸张、衬线大标题和报纸分栏，但这属于当前常见生成式设计默认值，且与“持续学习、工程验证”联系较弱，因此改为冷调工作台、真实状态信息和提交轨迹。下一轮仍需通过三个可比较的视觉方向验证这一概念，而不是直接把草案当成最终设计。
