# ADR-0001：采用 Vinext/Vite 的 Cloudflare-compatible 站点骨架

- 状态：Superseded by ADR-0003
- 日期：2026-07-18

## 背景

仓库只有初始 README，没有需要兼容的既有框架。项目需要多路由内容页面、可预览部署、独立域名和低维护成本。

## 决策

采用 Sites 提供的 Vinext 启动骨架：使用 React、Next.js 兼容 App Router、Vite 和 Cloudflare-compatible ESM 输出。内容保持仓库内管理，不启用 D1/R2，除非未来出现明确的持久化需求。

## 原因

- 生成结果与目标 Cloudflare 托管环境一致；
- 支持多页面、元数据和服务端组件组织方式；
- 不需要当前阶段维护数据库或独立服务器；
- 已包含构建、测试和托管声明的可运行基线。

## 代价

- 相比纯静态生成器，框架层更厚；
- Vinext 的兼容性需要通过每轮构建与浏览器验证持续确认；
- Markdown、搜索和 RSS 需要在此运行模型下显式实现。

## 被否决的当前替代方案

- Astro：内容体验优秀、静态输出简单，但本轮选定的 Sites/Cloudflare 交付流程提供了直接可部署骨架。若 Vinext 在内容构建或稳定性门槛上失败，将重新评估 Astro。
- 自建 Node 服务：对个人静态内容增加了不必要的运维面。

## 重新评估条件

- 关键 Next 路由或构建能力无法在 Vinext 中稳定工作；
- 内容发布需要运行时文件系统；
- 生产构建无法重复或 Cloudflare 部署长期不稳定。

## 结果

2026-07-19 作者选择不再使用 Cloudflare。项目已触发上述重新评估条件，并在 ADR-0003 迁移为 Vercel 原生 Next.js。本文件保留为首个公开版本的历史依据。
